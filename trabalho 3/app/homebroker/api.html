Descrição da API do Homebroker

  • Para GET e DELETE: Mandar argumentos como argumentos do request (ex: /recurso?arg=value)
  • Para POST: Mandar argumentos como JSON no corpo da mensagem

Error Codes

  1. SUCCESS
  2. CLIENT_ALREADY_EXISTS
  3. UNKNOWN_CLIENT
  4. NOT_ENOUGH_STOCK
  5. UNKNOWN_TICKER
  6. INVALID_MESSAGE
  7. FORBIDDEN_NAME

Recursos

  • /login:
    • GET:
      • args:
        • client_name
      • returns:
        • 200: EventStream
        • 400: INVALID_MESSAGE
        • 403: FORBIDDEN_NAME
        • 403: CLIENT_ALREADY_EXISTS
  • /status:
    • GET:
      • args:
        • client_name
      • returns:
        • 200: { "quotes": {ticker: quote}, "orders": {ticker: order}, "owned_stock": {ticker: amount}, "alerts": {ticker: [lower_limit, upper_limit]} }
        • 400: INVALID_MESSAGE
        • 404: UNKNOWN_CLIENT
  • /order:
    • POST:
      • args:
        • client_name
        • type
        • ticker
        • amount
        • price
        • expiry_date
        • active
      • returns:
        • 200: SUCCESS
        • 400: INVALID_MESSAGE
        • 400: EXPIRED_ORDER
        • 403: NOT_ENOUGH_STOCK
        • 404: UNKNOWN_CLIENT
        • 404: UNKNOWN_TICKER
  • /limit:
    • POST:
      • args:
        • ticker
        • lower_limit
        • upper_limit
        • client_name
      • returns:
        • 200: SUCCESS
        • 400: INVALID_MESSAGE
        • 404: UNKNOWN_CLIENT
        • 404: UNKNOWN_TICKER
  • /quote:
    • GET:
      • args:
        • client_name
      • returns:
        • 200: {ticker: quote}
        • 400: INVALID_MESSAGE
        • 404: UNKNOWN_CLIENT
    • DELETE:
      • args:
        • client_name
        • ticker
      • returns:
        • 200: SUCCESS
        • 400: INVALID_MESSAGE
        • 404: UNKNOWN_CLIENT
        • 404: UNKNOWN_TICKER
    • POST:
      • args:
        • client_name
        • ticker
      • returns:
        • 200: SUCCESS
        • 400: INVALID_MESSAGE
        • 404: UNKNOWN_CLIENT
        • 404: UNKNOWN_TICKER
